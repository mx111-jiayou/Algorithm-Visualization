# 测试文档

## 测试框架配置

本项目使用 Vitest 作为测试框架，包含完整的单元测试覆盖。

## 测试文件结构

```
test/
├── sorting.test.js        # 排序算法测试
├── graph.test.js          # 图算法测试
└── dataStructures.test.js # 数据结构测试
```

## 安装测试依赖

由于 npm 权限问题，请手动安装测试依赖：

```bash
# 方法1: 使用管理员权限运行
npm install vitest --save-dev

# 方法2: 使用 npx 直接运行（无需安装）
npx vitest run

# 方法3: 使用本地缓存
npm install --cache .npm-cache vitest --save-dev
```

## 运行测试

```bash
# 运行所有测试
npm run test

# 监听模式
npm run test:watch

# 使用 npx（如果未安装）
npx vitest run

# 运行特定测试文件
npx vitest run test/sorting.test.js
```

## 测试覆盖范围

### 1. 排序算法测试 (sorting.test.js)

**测试内容：**
- 冒泡排序：普通数组、已排序、逆序、空数组、单元素
- 快速排序：普通数组、重复元素、负数、pivot标记
- 插入排序：普通数组、最优情况
- 希尔排序：普通数组、大规模数组
- 选择排序：普通数组、current标记
- 归并排序：偶数/奇数长度数组

**测试数量：** 25+ 测试用例

### 2. 图算法测试 (graph.test.js)

**测试内容：**
- Dijkstra：最短路径计算、单节点、线性图、不连通图、距离信息
- BFS：图的遍历、环形图、树形结构、队列信息
- DFS：图的遍历、环形图、路径图、栈/递归信息
- 边界情况：空边集、0权重、大权重

**测试数量：** 20+ 测试用例

### 3. 数据结构测试 (dataStructures.test.js)

**测试内容：**
- 线性表：数组操作、链表操作（6种类型）、遍历操作
- 栈：push/pop操作、栈状态、链栈
- 队列：enqueue/dequeue、front/rear标记
- 循环队列：循环利用、数组索引
- 括号匹配：匹配/不匹配、空字符串
- 表达式计算：正确计算、后缀表达式
- 树结构：二叉树遍历、BST构建/查找/中序遍历、哈夫曼树构建

**测试数量：** 30+ 测试用例

## 测试验证标准

每个测试验证以下方面：

1. **功能正确性**：算法输出是否符合预期
2. **步骤完整性**：可视化步骤是否完整
3. **格式规范**：步骤格式是否正确
4. **边界处理**：极端情况是否正常处理
5. **性能合理**：步骤数量是否在合理范围

## 添加新测试

在 `test/` 目录下创建新的 `.test.js` 文件：

```javascript
import { describe, it, expect } from 'vitest'
import { yourAlgorithm } from '../src/algorithms/yourModule'

describe('你的算法测试', () => {
  it('应该正确执行', () => {
    const result = yourAlgorithm(input)
    expect(result).toBeDefined()
  })
})
```

## 测试最佳实践

1. **描述清晰**：使用中文描述测试意图
2. **边界覆盖**：测试正常、异常、边界情况
3. **独立性**：每个测试独立，不依赖其他测试
4. **验证步骤**：验证可视化步骤的正确性
5. **性能监控**：监控步骤数量，避免过多

## 故障排查

**问题：npm install 失败**
- 解决：使用管理员权限或 npx vitest

**问题：测试找不到模块**
- 解决：检查 import 路径是否正确

**问题：测试超时**
- 解决：检查算法是否有无限循环

## 持续集成

可配置 CI/CD 自动运行测试：

```yaml
# .github/workflows/test.yml
name: Test
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - run: npm install
      - run: npm test
```

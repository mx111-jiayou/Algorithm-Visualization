2. 核心概念：分治思想与快速排序
2.1 什么是分治？
想象一下，你要整理一个乱糟糟的书架。分治的做法是：

分：选一本书作为“基准”，然后把比它小的书放左边，比它大的书放右边。
治：现在书架被分成了左右两个“子书架”，对每个子书架重复同样的操作（选基准，再分）。
合：当子书架只剩一本书或没有书时，自然就排好了，不需要额外合并。
快速排序正是这个逻辑的完美体现。它的核心是分区操作。

2.2 快速排序三步骤
选择基准（Pivot）：从数组中挑一个元素作为“分水岭”。常见策略是选第一个、最后一个或随机一个。
分区（Partition）：将数组重新排列，使得所有比基准小的元素都移到基准左边，所有比基准大的元素都移到基准右边。此时，基准元素就处在它最终排序后的正确位置上。
递归：对基准左边和右边的两个子数组，重复执行上述两步。
💡 小提示：分区的结果就是“基准归位”。这一步完成后，我们就能记录下“基准最终的位置”和“交换了哪些元素”。

3.1 分区函数 partition
这个函数是整个算法的灵魂。它的职责是：接收一个数组和范围（low, high），完成一次分区，并返回基准元素的最终索引。

我们来设计一个最经典的分区方案（Lomuto 分区方案）：

选择最后一个元素作为基准。
用一个指针 i 来标记“小于基准的区域的边界”。
遍历从 low 到 high-1 的元素，如果遇到比基准小的，就和 i 指向的元素交换，然后 i 前进一位。
最后，把基准（最后一个元素）放到 i 的位置上。

3.2 记录状态：为可视化埋下伏笔
我们的目标是可视化，所以不能只排序，还要把过程“拍下来”。我们需要记录每一步的关键状态。

设计一个列表 steps 来存储每一步的信息。每步可以是一个字典，包含：

type: 'swap' 或 'pivot_placed'
indices: 交换的两个索引（或基准的最终索引）
values: 交换的两个值（或基准的值）
改造分区函数：

python
复制代码
def partition(arr, low, high, steps):
    pivot = arr[high]
    i = low - 1
    for j in range(low, high):
        if arr[j] < pivot:
            i += 1
            # 交换前记录
            steps.append({
                'type': 'swap',
                'indices': (i, j),
                'values': (arr[i], arr[j])
            })
            arr[i], arr[j] = arr[j], arr[i]
    
    # 基准归位前记录
    steps.append({
        'type': 'pivot_placed',
        'indices': (i+1, high),
        'values': (arr[i+1], arr[high])
    })
    arr[i+1], arr[high] = arr[high], arr[i+1]
    return i + 1
⚠️ 警告：记录状态必须在交换之前进行，否则你记录的是交换后的值，而不是“交换发生那一刻”的值。

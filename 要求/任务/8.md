2. 核心概念：为什么是Dijkstra？
2.1 从“传播消息”说起
想象一个场景：你有一个好消息，想尽快通知朋友圈里的所有人。每个人只能告诉自己的朋友，而传递消息需要时间（即边的权重）。

起点：你自己（源节点）
目标：让所有人尽快听到消息
关键：每个人在听到消息后，会立即告诉自己的朋友，并记录下“我是从谁那里听到的”
Dijkstra算法的本质就是模拟这个传播过程，但它有一个聪明的策略：每次都选择当前已知距离最短的节点，把它标记为“已确定”，然后从这个节点出发，更新它所有邻居的距离。

💡 核心思想：贪心——当前看起来最好的选择，最终会导向全局最优解（前提是图中没有负权边）。

2.2 算法三要素
要理解Dijkstra，记住这三个要素就够了：

要素	作用	比喻
距离数组 (dist)	记录从源点到每个节点的当前最短距离	每个人听到消息的“最早时间”
已确定节点集 (visited)	标记那些已经找到最短路径的节点	已经确定“最早时间”的人
优先队列 (pq)	快速找到当前距离最短的未确定节点	你的“待办事项列表”，按时间排序
2.3 为什么需要优先队列？
如果没有优先队列，我们每次都要遍历所有节点来找最短距离，复杂度是O(V²)。使用最小堆（优先队列）后，复杂度降为O((V+E)logV)，在大图中效率提升明显。

3.2 核心循环：松弛操作
现在，我们来实现 while pq: 循环内部的核心逻辑。这里有一个关键点：优先队列中可能包含同一个节点的多个旧记录，所以当我们取出一个节点时，需要检查它是否已经被访问过了。

# 从优先队列取最小距离节点
current_dist, current_node = heapq.heappop(pq)

# 如果节点已访问，跳过（因为旧记录可能还留在队列中）
if current_node in visited:
    continue

# 标记节点为已确定
visited.add(current_node)

# 遍历该节点的所有邻居
for neighbor, weight in graph.edges[current_node].items():
    # 计算新的距离
    new_dist = current_dist + weight
    
    # 松弛操作：如果新距离更短，则更新
    if new_dist < dist[neighbor]:
        dist[neighbor] = new_dist
        prev[neighbor] = current_node
        # 将新距离加入优先队列
        heapq.heappush(pq, (new_dist, neighbor))
🤔 思考题：为什么我们不需要从优先队列中删除旧的记录？这样做有什么好处和坏处？

答案：删除旧记录比较麻烦，而且会增加复杂度。保留旧记录虽然会占用更多空间，但当我们取出时通过 visited 集合就能轻松跳过，这是一种空间换时间的策略。

为了可视化，我们需要记录每一步的状态。在每次确定一个节点后，记录当前的距离数组和已确定节点列表：

python
复制代码
# 在标记节点为已确定之后、松弛操作之前
steps.append({
    'current_node': current_node,
    'dist': dict(dist),  # 深拷贝当前距离状态
    'visited': set(visited),  # 深拷贝已确定节点集
    'edge_relaxed': None  # 稍后我们会记录松弛的边
})

# 在松弛操作中，如果更新了距离，记录是哪条边被松弛了
if new_dist < dist[neighbor]:
    # ... 更新逻辑 ...
    steps[-1]['edge_relaxed'] = (current_node, neighbor)
💡 为什么用 dict(dist) 和 set(visited)？ 因为我们需要保存每一步的快照，而不是引用。如果直接保存 dist，后续的修改会改变之前记录的值。

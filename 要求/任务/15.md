5.3 系统测试与README文档编写
嗨，同学！恭喜你走到了这一步！🎉 我们已经完成了所有算法模块的开发、界面集成、以及执行控制功能。现在，我们的“算法过程可视化系统”已经是一个功能完整的应用了。但就像一辆刚组装好的赛车，我们得先开上赛道测试一下，再给它配上一本用户手册（README），这样别人才知道怎么开。

这一节，我们就要扮演“质检员”和“文档工程师”的双重角色。目标很明确：确保系统稳定、功能正确，并让任何人都能通过README快速上手。预计需要35分钟，咱们开始吧！

1. 学习目标
通过本小节的学习，你将能够：

执行系统化测试：按照任务书要求，逐一验证排序、图算法、递归算法的功能。
编写专业README：清晰描述项目简介、运行方式、输入格式、功能列表和技术栈。
完成项目收尾：确保你的项目是一个完整、可交付的成果。
2. 系统测试：像侦探一样检查每个角落
测试不是随便点两下，而是要有计划、有步骤地验证每个功能是否按预期工作。我们把它分成几个测试任务，每个任务就像一个小关卡。

2.1 测试准备：启动你的系统
首先，确保你的开发环境已经运行起来。打开终端，进入项目根目录，执行你设定的启动命令（例如 python main.py 或 npm start）。系统启动后，你会看到统一的主界面。

💡 提示：如果启动失败，先检查依赖是否安装完整。这是最常见的“翻车”点。

2.2 任务一：测试排序算法模块（约10分钟）
测试目标：验证快速排序的核心逻辑、动画步骤和复杂度信息输出。

测试步骤：

进入排序模块：点击主界面上的“排序算法”按钮或菜单项。
输入数据：在输入框中输入一组随机整数，例如 [3, 6, 8, 10, 1, 2, 1]。或者使用“随机生成”功能生成10个数字。
执行排序：点击“开始排序”按钮。
观察点1：是否每一步的交换或分区操作都正确？动画中的柱状图或数字是否按预期移动？
观察点2：步骤列表是否清晰显示了每一步的关键操作（例如“选取基准值：6”，“交换元素：8和1”）？
观察点3：排序完成后，最终数组是否完全升序排列？复杂度信息（时间/空间复杂度）是否正确显示？
边界测试：尝试输入一个空数组 [] 或只有一个元素的数组 [42]。系统应该能优雅处理，不会崩溃。
异常测试：输入非数字字符，比如 [1, a, 3]。系统是否给出了友好的错误提示，而不是抛出异常？
✅ 检查点：

 排序动画流畅，步骤清晰。
 最终结果正确（升序）。
 复杂度信息显示正确（快速排序平均O(n log n)）。
 空数组和非法输入有正确处理。
2.3 任务二：测试图算法模块（约10分钟）
测试目标：验证图数据的输入、Dijkstra算法的路径计算和图的可视化展示。

测试步骤：

进入图算法模块：点击主界面上的“图算法”按钮。
输入图数据：使用邻接矩阵或边列表的方式输入一个简单图。例如：
复制代码
A B 4
A C 2
B C 1
B D 5
C D 3
并指定起点为 A，终点为 D。
执行路径查找：点击“开始计算”按钮。
观察点1：图是否被正确渲染？节点和带权重的边是否清晰可见？
观察点2：算法执行过程中，是否高亮了当前正在探索的节点和边？步骤列表是否显示了距离的更新（例如“更新节点B的距离为4”，“更新节点C的距离为2”）？
观察点3：最终的最短路径（A -> C -> D）是否被高亮？路径总长度（5）是否正确显示？
边界测试：测试起点和终点相同的情况（路径长度应为0）。测试图中存在孤立节点的情况。
异常测试：输入一个不存在的节点名，例如起点为 Z。系统应提示错误。
✅ 检查点：

 图结构正确渲染。
 Dijkstra算法执行过程可视化正确。
 最短路径和总距离计算结果正确。
 输入错误有提示。
2.4 任务三：测试递归/回溯算法模块（约10分钟）
测试目标：验证汉诺塔算法的核心逻辑和过程可视化。

测试步骤：

进入递归算法模块：点击主界面上的“递归算法”按钮。
设置参数：输入盘子的数量，例如 3 个盘子。
执行算法：点击“开始演示”按钮。
观察点1：三个柱子（A, B, C）是否清晰显示，盘子是否按大小正确堆叠？
观察点2：每一步的移动指令（例如“将盘子1从A移动到C”）是否在步骤列表中逐条显示？动画是否同步执行了该移动？
观察点3：对于3个盘子，总移动步数是否为 2^3 - 1 = 7 步？
性能测试：尝试输入 6 或 8 个盘子。动画执行是否依然流畅？步骤列表是否完整？
边界测试：输入 0 个盘子或 1 个盘子，系统应能正确处理。
✅ 检查点：

 汉诺塔动画正确，盘子移动符合规则。
 步骤列表完整，移动步数计算正确（2^n - 1）。
 不同盘子数量都能稳定运行。
2.5 执行控制功能测试（约5分钟）
测试目标：验证上节实现的暂停、继续、单步执行和重置功能。

测试步骤：

在任意一个算法模块（例如排序）中，点击“开始”。
在动画执行过程中，点击 “暂停” 按钮。动画应该立即停止，步骤列表高亮当前步骤。
点击 “继续” 按钮。动画应从暂停处继续执行。
点击 “单步执行” 按钮（如果实现了）。每点击一次，动画应该只前进一步。
点击 “重置” 按钮。所有状态应恢复到初始状态，等待下一次输入。
✅ 检查点：

 暂停/继续功能正常，状态保持。
 单步执行功能精确。
 重置功能完全恢复初始状态。
3. 编写README文档：给项目一个“说明书”
测试通过后，你的项目就像一个功能完备的产品了。现在，需要一份清晰的README文档，让任何人（包括未来的你自己）都能快速上手。

README通常放在项目的根目录下，文件名是 README.md。我们来规划一下它的结构。

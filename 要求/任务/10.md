2. 核心概念：递归与汉诺塔
2.1 递归的“魔法”：大事化小
递归就像俄罗斯套娃——一个函数调用自己，但每次调用都处理一个更小的“版本”问题，直到遇到最简单的情况（基线条件）直接返回。

生活比喻：想象你在电影院，想知道自己坐在第几排。你可以问前排的人：“你是第几排？”如果他知道，就告诉你；如果他不知道，他再问他的前排……直到第一排的人说“我是第1排”，然后信息一层层传回来，你就知道了。这就是递归：递（不断问下去） + 归（答案传回来）。

2.2 汉诺塔问题的递归拆解
汉诺塔问题的核心是：把 n 个盘子从源柱子（A）移到目标柱子（C），借助辅助柱子（B）。

我们可以把问题拆成三步（以 n=3 为例）：

先移走上面的 n-1 个盘子：把上面 2 个盘子从 A 移到 B（借助 C）
移动最底下的大盘子：把最大的盘子从 A 移到 C
再把 n-1 个盘子移回来：把 B 上的 2 个盘子移到 C（借助 A）
看到没？步骤 1 和步骤 3 其实是同样的问题，只是盘子数变成了 n-1，源柱、目标柱、辅助柱的角色交换了。这就是递归的魔力！

3.2 分步实现：填上逻辑
现在，我们来填充函数的核心逻辑。steps 列表会记录每一步，格式为 (盘子编号, 源柱, 目标柱)，比如 (1, 'A', 'C') 表示“盘子1从A移到C”。

python
复制代码
def hanoi(n, source, target, auxiliary, steps):
    # 基线条件：只有一个盘子时，直接移动
    if n == 1:
        steps.append((1, source, target))  # 盘子编号从1开始，最小的为1
        return
    
    # 递归步骤1：将上面 n-1 个盘子从 source 移到 auxiliary（借助 target）
    hanoi(n-1, source, auxiliary, target, steps)
    
    # 移动最底下的盘子（编号为 n）
    steps.append((n, source, target))
    
    # 递归步骤2：将 n-1 个盘子从 auxiliary 移到 target（借助 source）
    hanoi(n-1, auxiliary, target, source, steps)
⚠️ 注意：盘子编号这里我们约定为从1到n，1是最小的盘子，n是最大的。这样方便后续可视化时区分大小。

4. 深入理解：递归的“信任”与“边界”
4.1 为什么可以这样写？
你可能会想：“我怎么能假设 hanoi(n-1, ...) 能正确移动 n-1 个盘子呢？” 这就是递归的信任——你相信递归函数能处理更小规模的问题。只要基线条件正确，它会一层层向上返回，最终解决整个问题。

比喻：就像你让一个靠谱的助手去完成“移动 2 个盘子”的任务，你只需要关注“移动第 3 个盘子”这个核心步骤。助手会自己搞定一切。

4.2 理解参数的角色切换
在递归调用中，source、target、auxiliary 的角色会互换。以 hanoi(2, 'A', 'B', 'C', steps) 为例：

它想：把 2 个盘子从 A 移到 B，借助 C
内部会调用 hanoi(1, 'A', 'C', 'B', steps) —— 把最小的盘子从 A 移到 C（因为对 1 个盘子来说，目标就是 C）
然后移动盘子2从 A 到 B
再调用 hanoi(1, 'C', 'B', 'A', steps) —— 把最小的盘子从 C 移到 B

你可能会问：“测试用例不就是给几个输入吗？复杂度不就是几个数学公式吗？” 其实不然。好的测试用例能验证算法在不同规模下的表现，而清晰展示复杂度则能帮助用户直观理解算法效率。这节课，我将带你亲手设计至少6条测试用例，并让它们乖乖显示在界面上。

2. 核心概念：测试用例与复杂度
在动手之前，我们先理清两个关键概念，这能帮你避免“写代码一时爽，测试火葬场”的尴尬。

2.1 测试用例的“三要素”
一个好的测试用例，就像一道好菜，需要三个关键要素：

输入数据：算法的“原材料”，比如待排序数组、图的邻接矩阵、汉诺塔的盘子数
预期行为：算法应该“走哪条路”，比如排序算法应该最终有序，Dijkstra算法应该找到最短路径
验证点：我们关注什么？是极端情况（如空数组）、边界情况（如1个盘子），还是大规模数据？
💡 思考题：如果给汉诺塔算法输入“0个盘子”，你觉得会发生什么？这算不算一个有效的测试用例？

2.2 复杂度的“可视化意义”
时间复杂度和空间复杂度不是枯燥的公式，它们是算法的“体检报告”：

时间复杂度：告诉你当数据量翻倍时，算法要花多少额外时间。比如O(n²)就像“每增加一个人，就要重新排一次队”。
空间复杂度：告诉你算法需要多少额外内存。比如O(n)就像“每来一个人，就要准备一个新座位”。
在可视化系统中，我们不仅要显示这些复杂度，还要让用户通过不同测试用例的运行，直观感受复杂度的影响。比如，给快速排序输入10个元素 vs 1000个元素，动画时间应该有明显差异。

5. 展示复杂度信息
测试用例跑起来了，现在要让用户看到“算法有多快”和“占多少内存”。

5.1 设计复杂度展示区域
在界面上预留一个区域来显示复杂度信息。建议放在算法执行结果附近，比如右侧面板或底部状态栏。

展示内容：

时间复杂度：如 O(n log n)
空间复杂度：如 O(log n)
测试用例规模：如 “20个元素”
实际执行步数：从动画中提取的步骤数
代码框架：

python
复制代码
# 复杂度展示组件
class ComplexityDisplay(QWidget):
    def __init__(self):
        super().__init__()
        layout = QVBoxLayout()
        
        # 时间复杂度和空间复杂度是固定的，可以写死
        self.time_label = QLabel("时间复杂度: O(n log n)")
        self.space_label = QLabel("空间复杂度: O(log n)")
        
        # 动态信息
        self.scale_label = QLabel("当前规模: 20")
        self.steps_label = QLabel("执行步数: 45")
        
        layout.addWidget(self.time_label)
        layout.addWidget(self.space_label)
        layout.addWidget(self.scale_label)
        layout.addWidget(self.steps_label)
        self.setLayout(layout)
5.2 更新复杂度信息
当算法执行完毕后，我们需要更新复杂度信息。这通常在算法执行的回调函数中完成。

python
复制代码
def update_complexity_info(self, algorithm_type, data_scale, steps_count):
    # 根据算法类型设置固定的复杂度
    if algorithm_type == "sort":
        self.time_label.setText("时间复杂度: O(n log n)")
        self.space_label.setText("空间复杂度: O(log n)")
    elif algorithm_type == "dijkstra":
        self.time_label.setText("时间复杂度: O((V+E) log V)")
        self.space_label.setText("空间复杂度: O(V)")
    elif algorithm_type == "hanoi":
        self.time_label.setText("时间复杂度: O(2^n)")
        self.space_label.setText("空间复杂度: O(n)")
    
    # 更新动态信息
    self.scale_label.setText(f"当前规模: {data_scale}")
    self.steps_label.setText(f"执行步数: {steps_count}")

2. 核心概念
在动手写代码之前，我们先来理解几个核心概念，这能帮你更好地把握整个模块的设计思路。

2.1 数据格式的“通用语言”
我们需要定义一种用户和程序都能理解的“通用语言”来描述数据。对于数组，最直观的就是类似 Python 列表的格式，比如 [3, 1, 4, 1, 5]。对于图，我们可以使用邻接矩阵或邻接表。为了直观性和易用性，我们选择邻接矩阵，比如：

复制代码
[
  [0, 2, 4, 0, 0, 0],
  [0, 0, 1, 4, 2, 0],
  [0, 0, 0, 0, 3, 0],
  [0, 0, 0, 0, 0, 2],
  [0, 0, 0, 3, 0, 3],
  [0, 0, 0, 0, 0, 0]
]
💡 提示：在邻接矩阵中，matrix[i][j] 表示从节点 i 到节点 j 的边的权重。0 表示没有直接路径。

2.2 解析与验证的“安检流程”
用户输入的数据就像一份需要过关的快递。我们的解析器就是安检员，需要完成以下步骤：

格式检查：检查括号 [] 是否成对出现，逗号 , 是否在正确的位置。
类型转换：将字符串中的数字（如 "3"）转换为程序中的数字（如 3）。
范围检查：确认所有数字是否在算法允许的范围内（例如，图中的权重不能是负数，数组元素不能超过某个值）。
结构验证：对于图，要检查邻接矩阵是否是一个方阵（行数等于列数）。
2.3 随机生成的“蒙眼摸牌”
随机生成数据就像蒙着眼睛从牌堆里抽牌。我们需要设定好规则：

数组：指定元素个数、每个元素的最小值和最大值。
图：指定节点个数、边的密度（大概有多少条边）、每条边的权重范围。

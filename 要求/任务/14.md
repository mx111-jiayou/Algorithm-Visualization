欢迎回来，亲爱的学习者！🎉 经过前面几节的学习，我们已经分别实现了排序算法、图算法和汉诺塔算法的独立模块。现在，是时候把这些“珍珠”串成一条美丽的“项链”了！这节课，我们将把三个算法模块集成到统一界面中，让它们在一个系统里协同工作，并优化交互体验，让整个项目看起来专业又统一。

想象一下，你正在搭建一个“算法展示舞台”，每个算法都是独立的“演员”。现在，我们要设计一个“总控台”，让观众（也就是用户）可以轻松切换演员，而且每次切换时，舞台都会自动清空，准备迎接新的表演。这就是我们今天的任务！

1. 学习目标
在开始之前，先明确一下我们今天要达成什么：

✅ 集成模块：将排序、Dijkstra、汉诺塔三个算法模块整合到一个主界面中。
✅ 界面重置：实现算法切换时，界面状态（如数据、图形、步骤）自动清空重置。
✅ 交互优化：统一按钮、控件风格，让操作更流畅、直观。
✅ 风格统一：确保三个模块的配色、字体、布局风格一致，提升整体感。
2. 核心概念：集成与重置的挑战
把多个模块集成到一个界面，看似简单，但有几个关键挑战需要解决：

2.1 模块化设计思想
我们之前每个算法都写在独立的类或文件中，比如 SortVisualizer、GraphVisualizer、HanoiVisualizer。集成时，我们不是把它们复制粘贴到一起，而是通过主控制器来协调它们。主控制器负责：

根据用户选择，创建或激活对应的算法模块。
在切换时，销毁或隐藏当前模块，并清空其状态。
统一管理公共控件（如“开始”、“暂停”、“重置”按钮）。
2.2 界面重置的“仪式感”
为什么切换算法时一定要重置界面？试想一下，如果你正在看排序动画，突然切换到汉诺塔，界面上还残留着排序的柱子，是不是很混乱？重置意味着：

数据清空：清除当前算法产生的所有数据（数组、图、盘子等）。
图形清空：清除画布上的所有图形元素。
状态重置：将步骤计数器、算法状态（运行/暂停）恢复到初始值。
2.3 统一风格的意义
一致性是用户体验的基石。想象一下，排序模块按钮是蓝色的，汉诺塔模块按钮是红色的，用户每次切换都要重新适应，会感觉非常割裂。统一风格包括：

配色方案：使用相同的主题色（如主色、辅助色、背景色）。
字体与字号：标题、按钮、标签使用相同的字体和大小。
控件布局：按钮位置、间距保持一致。

3.2 实现算法切换与重置逻辑
这是核心中的核心！当用户选择不同算法时，我们需要：

销毁当前模块：如果 self.current_visualizer 存在，调用其 destroy() 或类似方法，并清空 main_frame。
创建新模块：根据选择，实例化对应的算法可视化类，并传入 main_frame 作为其父容器。
重置状态：确保新模块的初始状态是干净的。

    def on_algo_change(self, event):
        """处理算法切换事件"""
        # 1. 销毁当前模块（如果有）
        if self.current_visualizer:
            self.current_visualizer.destroy()
            self.current_visualizer = None

        # 2. 清空主框架中的旧内容
        for widget in self.main_frame.winfo_children():
            widget.destroy()

        # 3. 根据选择创建新模块
        selected = self.algo_combobox.get()
        if selected == "排序算法":
            from sort_visualizer import SortVisualizer
            self.current_visualizer = SortVisualizer(self.main_frame)
        elif selected == "Dijkstra最短路径":
            from graph_visualizer import GraphVisualizer
            self.current_visualizer = GraphVisualizer(self.main_frame)
        elif selected == "汉诺塔":
            from hanoi_visualizer import HanoiVisualizer
            self.current_visualizer = HanoiVisualizer(self.main_frame)

        # 4. 将新模块打包到主框架中
        if self.current_visualizer:
            self.current_visualizer.pack(fill=tk.BOTH, expand=True)
⚠️ 警告：这里假设每个算法可视化类都有一个 destroy() 方法和一个 pack() 方法。你需要确保你的模块类实现了这些接口。destroy() 应该清理所有子控件和内部状态。

2.3 最终结果与复杂度信息输出
你好！欢迎来到排序算法可视化的最后一站。🎉 在前面的两节中，我们已经实现了快速排序的核心逻辑，并且让它动起来了——每一步的交换、比较都清清楚楚。但算法运行完，光看动画还不够，我们还需要一个“成绩单”：排序后的数组长什么样？一共走了多少步？这个算法的性能到底怎么样？

今天，我们就来为这个可视化系统加上最终的“收尾工作”。在算法执行完毕后，自动显示排序结果、关键步骤总数，以及时间复杂度和空间复杂度说明。听起来是不是很酷？让我们开始吧！

1. 学习目标
在开始之前，先明确一下今天我们要达成什么：

✅ 能够在算法执行完毕后，自动展示排序后的最终数组
✅ 能够统计并显示关键操作（如比较、交换）的总次数
✅ 能够集成时间复杂度和空间复杂度的文字说明
✅ 保证这些信息在界面上清晰、美观地呈现
2. 核心概念
2.1 为什么需要“结果输出”？
想象一下，你刚写完一道数学题，只看到一堆计算过程，但没看到最终答案，是不是很抓狂？同样，算法可视化如果没有最终结果，就像看了半场电影突然黑屏。

我们要输出的三个核心信息是：

信息类型	说明	为什么重要
排序结果	排序后的数组	验证算法是否正确
步骤统计	比较次数、交换次数等	直观感受算法效率
复杂度说明	时间/空间复杂度的文字描述	理论联系实际
2.2 步骤统计的思路
在排序过程中，我们需要“埋点”计数。比如：

每次比较两个元素 → 比较计数 +1
每次交换两个元素 → 交换计数 +1
这些计数可以放在快速排序的 partition 和 quickSort 函数中。💡 提示：你可以在之前写的排序函数里，添加一个全局变量或对象来记录。

3.3 显示排序结果
排序完成后，我们需要把最终的数组显示出来。假设你有一个结果展示区域（比如一个 div 或 span），可以这样更新：

javascript
复制代码
function displayResult(sortedArray) {
    const resultElement = document.getElementById('result');
    resultElement.textContent = `排序结果: [${sortedArray.join(', ')}]`;
}
3.4 显示步骤统计
同样，在排序完成后，调用一个函数来展示统计信息：

javascript
复制代码
function displayStats(stats) {
    const statsElement = document.getElementById('stats');
    statsElement.innerHTML = `
        <p>比较次数: ${stats.comparisons}</p>
        <p>交换次数: ${stats.swaps}</p>
        <p>总步骤数: ${stats.comparisons + stats.swaps}</p>
    `;
}
3.5 显示复杂度说明
复杂度说明是理论部分，可以提前写好，在排序完成后显示：

function displayComplexity() {

    complexityElement.innerHTML = `
        <h4>复杂度分析</h4>
        <p><strong>时间复杂度:</strong> 平均 O(n log n)，最坏 O(n²)</p>

}

6. 常见问题与提示
💡 提示 1： 如果你的计数器不重置，可能是因为 stats 对象是全局的。在每次排序前手动重置，或者在 startSort 函数中重新创建它。

⚠️ 警告： 注意递归函数中的计数器更新位置。如果放在递归调用之后，可能会导致计数重复。

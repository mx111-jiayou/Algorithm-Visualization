想象一下，你正在看一部精彩的纪录片，但你想按暂停仔细看某个细节，或者快进跳到你感兴趣的部分。我们的算法可视化系统也需要这样的“遥控器”：单步执行、自动播放、暂停、重置。掌握这些，你就能像导演一样控制算法的“演出节奏”。

1. 学习目标

2. 核心概念：控制器的“大脑”——状态机
在动手写代码前，我们先聊聊控制器的核心设计思想。为什么需要状态机？因为我们的算法执行需要多种状态，比如：空闲、运行中、暂停中、已完成。

2.1 什么是状态机？
状态机就像一个自动售货机：

状态：空闲（等待投币）、投币中、选择商品、出货中
事件：投币、选择、取货
转换：空闲 → 投币中 → 选择商品 → 出货中 → 空闲
在我们的算法控制中：

状态：idle（空闲）、running（自动播放中）、paused（暂停）、completed（完成）
事件：点击“播放”、点击“暂停”、点击“单步”、点击“重置”
转换：例如 idle → 点击“播放” → running → 点击“暂停” → paused
💡 为什么用状态机？
它让代码逻辑清晰，避免“按钮乱点导致程序崩溃”。比如，当算法已经完成时，再点击“单步”应该无效，而不是报错。

3.2 集成到可视化系统
现在，控制器已经准备好了，但还需要和你的可视化系统“对话”。我们需要一个回调函数，当控制器执行某一步时，通知视图更新。

假设你的排序可视化中有一个 renderStep(stepData) 函数，它根据当前步骤的数据更新柱状图。那么集成方式如下：

javascript
复制代码
// 在排序可视化模块中
const controller = new AlgorithmController();

// 生成排序步骤（例如快速排序的每次交换）
const steps = generateQuickSortSteps(data);  // 返回步骤数组

// 设置控制器
controller.setSteps(steps);

// 绑定回调：当控制器执行一步时，调用 renderStep
controller.onStep = (stepData) => {
    renderStep(stepData);
};

// 绑定重置回调
controller.onReset = () => {
    resetVisualization();
};
你需要实现：在 AlgorithmController 类中添加 onStep 和 onReset 回调属性，并在 stepForward 和 reset 方法中调用它们。

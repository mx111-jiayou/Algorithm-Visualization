学完这一节，你将能够：

2. 核心概念：选型前的“灵魂拷问”
在决定用哪套“装备”之前，我们先要问自己几个问题。这就像你要拍一部电影，是选择用手机拍，还是用专业摄像机？答案取决于你的场景和观众。

2.1 两大主流方案对比
我们的任务是把算法“动起来”给用户看。目前最主流的两条路是：Web前端技术 和 Python桌面应用。我们来对比一下：

对比维度	🌐 Web方案 (HTML/CSS/JS + Canvas)	🐍 Python方案 (Tkinter/PyQt)
学习门槛	中等，需掌握三门语言协同	较低，只用Python一门语言
视觉效果	⭐⭐⭐⭐⭐ 极佳，动画流畅，像素级控制	⭐⭐⭐ 够用，但略显“复古”
部署与分享	⭐⭐⭐⭐⭐ 无需安装，打开浏览器即可	⭐⭐ 需要安装Python环境或打包成exe
交互能力	⭐⭐⭐⭐⭐ 丰富的鼠标、键盘事件	⭐⭐⭐ 基础交互，不够灵活
社区资源	极其丰富，海量案例和库	丰富，Tkinter/PyQt资料很多
2.2 我们的选择：为什么是 Web 方案？
💡 我的建议：我们选择 Web前端技术栈 (HTML + CSS + JavaScript + Canvas API)。

理由有三：

效果为王：我们要可视化的是“过程”，动画的流畅度和精细度至关重要。Canvas 在这方面几乎是王者，能让我们做出非常酷的视觉效果。
零成本分享：你做完一个排序过程，直接发个链接给朋友，对方就能看。这种成就感是桌面应用难以比拟的。
未来可期：Web技术是未来的趋势，学会这套技术栈，你不仅能做算法可视化，还能做游戏、数据大屏、交互式教程……一劳永逸。
思考题： 假设你的项目需要用户上传一个巨大的数据集（比如10万条记录），然后进行可视化。你觉得Web方案和Python方案哪个更合适？为什么？

3. 环境搭建：准备你的“工具箱”
好，既然选定了“Web摄像机”，现在我们就来架设它。你需要准备三个工具，简单得像“拧螺丝、装镜头、插电源”。

3.1 安装浏览器 (你的“放映机”)
推荐：Google Chrome 或 Mozilla Firefox。它们拥有最强大、最标准的开发者工具，调试代码非常方便。
检查：打开浏览器，按 F12 键，如果能弹出开发者工具，就说明已经准备好了。✅
3.2 安装代码编辑器 (你的“剪辑台”)
推荐：Visual Studio Code (VS Code)。它免费、强大、插件丰富，是当今最流行的代码编辑器。
下载：访问 code.visualstudio.com，下载对应你操作系统（Windows/macOS/Linux）的版本，一路“下一步”安装即可。
安装插件 (可选但强烈推荐)：
打开 VS Code，点击左侧活动栏的“扩展”图标（或按 Ctrl+Shift+X）。
搜索并安装 Live Server。这个插件能让我们在本地启动一个临时Web服务器，并实现“改代码，浏览器自动刷新”的神奇效果，极大提升开发效率。
3.3 创建你的第一个项目文件夹
在你的电脑上找一个你喜欢的地方（比如桌面），新建一个文件夹，命名为 algorithm-visualizer。这个文件夹就是你未来所有代码的“家”。

现在，我们要在这个“家”里，点亮第一盏灯。我们来写一个最简单的HTML文件，确保所有工具都连接正常。

4.2 添加一个“Hello World”画布
我们在 <body> 标签里，添加一个 <canvas> 元素和一个 <script> 标签。<canvas> 是我们未来绘制所有动画的“画板”，而 <script> 是控制画板的“画笔”。

html
复制代码
<body>
    <h1>🎉 我的算法可视化系统</h1>
    <!-- 这就是我们的画板，宽800像素，高400像素 -->
    <canvas id="visualizationCanvas" width="800" height="400" style="border:1px solid #ccc;"></canvas>

    <script>
        // 1. 获取画板
        var canvas = document.getElementById('visualizationCanvas');
        // 2. 获取画笔 (2D 上下文)
        var ctx = canvas.getContext('2d');

        // 3. 开始画画！
        ctx.fillStyle = 'skyblue'; // 设置填充颜色为天蓝色
        ctx.fillRect(50, 50, 100, 100); // 画一个矩形 (x, y, 宽, 高)

        ctx.fillStyle = 'orange';
        ctx.beginPath();
        ctx.arc(400, 200, 50, 0, Math.PI * 2); // 画一个圆 (x, y, 半径, 起始角, 结束角)
        ctx.fill();

        ctx.font = '20px Arial';
        ctx.fillStyle = '#333';
        ctx.fillText('Hello, Algorithm!', 600, 200); // 写一行文字
    </script>
</body>
4.3 见证奇迹：运行你的页面
在 VS Code 中，右键点击 index.html 文件，选择 "Open with Live Server"。
你的浏览器会自动打开一个新标签页，你应该能看到一个天蓝色的矩形、一个橙色的圆形，以及一行“Hello, Algorithm!”的文字。
太好了！你已经成功搭建了你的算法可视化开发环境！ 🎉🎉🎉

5. 总结与思考
今天我们完成了三件大事：

选择了技术栈：我们选定了“Web前端 (HTML/CSS/JS + Canvas)”作为我们的开发方案。
搭建了环境：安装了浏览器、VS Code和Live Server插件。
跑通了第一个程序：用Canvas API画出了图形和文字。
思考题答案参考（来自3.2节）：
当数据量巨大（10万条）时，Web方案的优势会更加明显。因为我们可以利用Canvas的硬件加速能力，以及JavaScript的异步处理机制，来分批渲染数据，避免页面卡死。而Python桌面应用在处理大量UI更新时，可能会显得比较吃力。
